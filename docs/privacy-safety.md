# Privacy and Safety Notes

Proudly. handles sensitive traveler context. Treat LGBTQIA+ profile preferences and precise location data as high-sensitivity information.

## Product rules

- Make identity fields optional.
- Default profiles to private.
- Never expose sexual orientation, gender identity, safety preferences, or trip plans publicly without explicit consent.
- Allow anonymous reviews.
- Allow users to delete their account and data.
- Do not promise any venue or destination is “100% safe.”
- Use language like Verified Safe, Community Rated, Inclusive Stay, Use Caution, and Not Currently Recommended.

## Safety profile model

A listing safety profile should include:

- Verification status.
- Safety score.
- Tags such as trans-friendly, LGBTQ-owned, POC-friendly, gender-neutral, wheelchair-accessible.
- Last reviewed date.
- Notes and source.
- Community review history.
- Reported incident moderation.

## Legal/ops recommendation

Before launch, get counsel to review privacy policy, terms, moderation rules, affiliate disclosures, and booking liability language.
